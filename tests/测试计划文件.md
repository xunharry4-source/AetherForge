# 测试计划 - Novel Agent 项目重构

## 测试目标
1. **证明正常情况稳定可用**: 验证重构后的项目目录结构下，后端 API (`app_api.py`) 能够正常启动并正确加载所有 Agent 和工具类。
2. **证明异常和特殊情况不会把系统拖入失控**: 验证当缺失配置文件或环境变量时，系统能正确报错并降级，而不是产生循环导入或静默失败。
3. **主动发现未知问题**: 通过全量扫描 `src/` 目录下的 Python 文件，检查是否存在遗漏的旧路径导入。
4. **为稳定性增强、回滚设计、降级设计提供证据**: 记录重构前后的性能差异（如有）及模块化带来的解耦收益。

## 项目强制证明项
- **功能真的正确**: 重构后的 Agent 逻辑（Worldview, Outline, Writing）在物理执行上与重构前一致。
- **页面展示来自真实结果**: 验证前端 API 调用返回的数据结构符合 React Flow 和 MUI 组件的要求。
- **失败时能正确降级**: 验证 RAG 检索在数据库不可用时能回退到本地 JSON 文件。

## Web 测试拆分
1. **前端访问接口测试**:
   - `GET /api/system/config`: 验证能够正确获取重构后的配置信息。
   - `POST /api/router/dispatch`: 验证路由 Agent 能够正确分发任务。
2. **前端页面功能测试**:
   - 验证 `Workflow.tsx` (React Flow) 能够渲染来自 `/api/lore/all` 的真实节点数据。
