## Authentication

Register and login return both a login token and a user API Key:

- Login token: generated with `secrets.token_urlsafe(32)` and stored in `auth_sessions.token`.
- User API Key: generated with `na_` prefix plus `secrets.token_urlsafe(32)` and stored in `users.api_key`.
- New users receive an API Key during `POST /api/auth/register`.
- Existing users without an API Key receive one during their next successful `POST /api/auth/login`.
- API Keys are returned as both top-level `api_key` and `user.api_key` in register/login responses.

```http
Authorization: Bearer <token>
```

External callers can skip login and access protected user endpoints with the user's API Key:

```http
X-API-Key: <api_key>
```

Alternative API Key header:

```http
Authorization: ApiKey <api_key>
```

Protected endpoints currently include `GET /api/auth/me`. `POST /api/auth/logout` only invalidates login tokens; API Keys remain valid after logout.

## auth

### `POST /api/auth/login`

- Endpoint: `login_user`
- Summary: login user
- OpenAPI path: `/api/auth/login`
- Auth note: Authenticates with `username` and `password`, then returns `token`, `api_key`, and `user`.
- Auth note: If an existing user has no `api_key`, login generates one with the same `na_` + `secrets.token_urlsafe(32)` format and persists it.
- Auth note: Required JSON fields: `username`, `password`.

### `POST /api/auth/logout`

- Endpoint: `logout_user`
- Summary: logout user
- OpenAPI path: `/api/auth/logout`
- Auth note: Invalidates the login token passed as `Authorization: Bearer <token>`.
- Auth note: API Key access does not depend on login sessions and remains valid after logout.

### `GET /api/auth/me`

- Endpoint: `get_current_user`
- Summary: get current user
- OpenAPI path: `/api/auth/me`
- Auth note: Requires either a login token or a user API Key.
- Auth note: Accepted headers: `Authorization: Bearer <token>`, `X-API-Key: <api_key>`, or `Authorization: ApiKey <api_key>`.

### `POST /api/auth/register`

- Endpoint: `register_user`
- Summary: register user
- OpenAPI path: `/api/auth/register`
- Auth note: Creates a user, stores a hashed password, generates a user API Key, creates a login session, and returns `token`, `api_key`, and `user`.
- Auth note: API Key generation: `na_` prefix plus `secrets.token_urlsafe(32)`, stored on the user document as `users.api_key`.
- Auth note: Required JSON fields: `username`, `password`. Optional JSON fields: `display_name`, `email`.
