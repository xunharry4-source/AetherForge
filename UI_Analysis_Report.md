## 2. 世界观知识库
### 页面说明
小说设定的核心存储与分级管理中心，支持对种族、势力、地理等实体进行树状化管理。

### 功能表格
| 功能 | 功能说明 | 是否需要后端 | 后端访问地址 | 是否已经完成 | 功能是否正常 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| 分类树构建 | 将扁平的世界观记录按类别层级组织成树状结构 | 是 | `/api/lore/tree` | ✅ 是 | ✅ 正常 |
| 实体详情预览 | 点击树节点展示实体的 Markdown/JSON 详细描述 | 是 | `/api/lore` | ✅ 是 | ✅ 正常 |
| 条目实时编辑 | 直接在浏览器中修改既有设定的内容并即时同步 | 是 | `/api/archive/update` | ✅ 是 | ✅ 正常 |
| 条目物理删除 | 从本地数据库及内存索引中永久移除指定条目 | 是 | `/api/archive/delete` | ✅ 是 | ✅ 正常 |
| 全局关键词搜索| 支持在知识库页面通过名称或内容实时检索实体 | 是 | `/api/search` | ✅ 是 | ⚠️ 测试无结果 |
| 实体关系图谱 | 动态生成并展示实体间的力导向关联关系图 | 是 | `/api/lore/entity-graph` | ✅ 是 | ✅ 正常 |
| 知识思维导图 | 以树状思维导图形式可视化世界观分类结构 | 是 | `/api/lore/mindmap` | ✅ 是 | ✅ 正常 |

> [!NOTE]
> **测试反馈 (2026-03-27)**: 知识检索功能在当前 UI 下测试返回“无结果”。
> **原因分析**: 现有的 UI 搜索条采用的是客户端**关键词过滤**（Keyword Filter），而用户期望可能是基于向量数据库的**语义搜索**（Semantic Search）。如果关键词不匹配或索引未完全加载，会导致无结果。
