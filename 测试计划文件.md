# API 临床级验证测试计划 (Requests 物理执行)

## 测试目标

1. **证明正常情况稳定可用**: 验证核心 API 路径（健康检查、列表获取、搜索、创建）在正常输入下返回 200 并包含正确的数据结构。
2. **证明异常和特殊情况不会把系统拖入失控**: 验证 API 在处理无效 ID、缺失参数、错误类型时能正确返回 400/404/500，且后台服务不崩溃。
3. **主动发现未知问题**: 通过真实的 HTTP 请求发现潜在的数据库连接、向量库同步或 JSON 解析错误。
4. **为稳定性增强、回滚设计、降级设计提供证据**: 记录响应时间、错误详情，为后续优化提供物理依据。

## 明确禁止项

- 禁止使用 Mock。
- 禁止只测成功路径。
- 禁止仅凭逻辑推演，必须进行真实的物理 HTTP 调用。

## 验证方法详述

- **准备**: 确保 API 服务器正在 `localhost:5005` 运行。
- **执行**: 运行 `python tests/clinical_api_requests_test.py`。
- **断言点**:
  - HTTP 状态码 (200, 400, 404 等)。
  - JSON 响应体中的 `status`, `error`, `id` 等关键字段。
  - 后台日志是否出现 Exception。
- **判定**: 若返回 500 且伴随堆栈，判定为产品缺陷；若连接失败，判定为环境问题。
